# Austin 的第二大脑 — 架构规划

## 背景

本项目是一个内网 Doc Hub，统一整合以下内容：

- 多个 AI 辅助编程项目（finance、growing、tennis、stock 等）
- 多个主题知识库（财富、求职等，未来持续扩展）
- 个人博客（技术向 + 个人向）
- 笔记体系（Notion + Obsidian）

部署在家庭 NAS 上，仅供内网访问。如需对外公开，另行处理。

---

## 技术选型

- **渲染框架**：Quartz（统一使用）
- **笔记编辑**：Obsidian（可单独打开各知识库的 wiki/ 目录）
- **代码托管**：GitHub（各 repo 独立管理）
- **部署环境**：NAS 内网

---

## 整体架构（方案 A：单 Quartz Hub）

### 核心思路

- **一个 Quartz 实例**作为 Hub，统一渲染所有内容，单服务、单入口
- **各知识库保持独立 git repo**，互不合并
- **NAS 上通过 sync 控制**哪些目录进入 Hub 的 content/，不同项目可灵活配置（不限于 wiki/）
- **output/ 永不离开本机**，不上 GitHub，不 sync 到 NAS

### Hub repo 目录结构

```
Austin的第二大脑/          ← Hub repo（本项目）
├── content/
│   ├── index.md           ← Hub 首页（导航入口）
│   ├── wealth/            ← 从 wealth-kb repo sync 过来
│   ├── job/               ← 从 job-kb repo sync 过来
│   ├── projects/          ← 直接写在 Hub repo（各项目介绍/链接）
│   └── blog/              ← 直接写在 Hub repo
├── quartz.config.ts
└── package.json
```

### 各知识库 repo 标准结构

```
<topic>-kb/
├── raw/       ← 原始材料（问题、素材、参考资料等）
├── wiki/      ← 知识库正文（Obsidian Vault，可单独打开）
└── output/    ← 含个人信息，.gitignore 排除，永不上传
```

NAS sync 策略：按项目灵活决定 sync 哪些目录到 Hub 的 content/ 对应子目录，通常为 wiki/，也可包含其他目录。

## 首页设计

### 导航结构

首页按**生活领域**分区，每张卡片带**内容类型标签**（知识库 / 编程项目 / 博客）。
领域决定怎么找它，类型决定技术上怎么处理它，两个维度独立。

| 领域 | 内容 | 类型 | 技术处理 |
|------|------|------|---------|
| 🏠 生活 | 财富知识库 | 知识库 | sync → content/wealth/，Quartz 渲染 |
| 🏠 生活 | Finance | 编程项目 | 卡片外链到 GitHub |
| 🏠 生活 | Stock | 编程项目 | 卡片外链（规划中）|
| 🏠 生活 | Immich 照片 | 媒体 | 卡片外链到 nas.local:2283，内容页可 link 共享相册 |
| 💼 工作 | 求职知识库 | 知识库 | sync → content/job/，Quartz 渲染 |
| 📖 学习 | AI / LLM 知识库 | 知识库 | sync → content/llm/，Quartz 渲染（即将建立）|
| 📖 学习 | Growing | 编程项目 | 卡片外链到 GitHub |
| 📖 学习 | 技术博客 | 博客 | 卡片外链到 GitHub Pages |
| 🎮 娱乐 | Tennis | 编程项目 | 卡片外链到 GitHub |
| 🎮 娱乐 | 个人博客 | 博客 | 待建 |

笔记（Notion、Obsidian）是外部系统，首页底部单独展示，不做可点击卡片。

### 照片集成方式（Immich）

NAS 上已有 Immich，照片不进 Quartz content，通过以下方式集成：
- Hub 首页生活区放 Immich 入口卡片，直链内网地址
- wiki 或生活内容页中，用 Immich 共享相册功能生成链接，在 markdown 里直接引用
- 无需额外配置，不占 Quartz 存储
